# Architecture

## Project

Взят за основу Defect Dojo
DefectDojo - это бесплатная платформа для управления уязвимостями (класса ASPM) с открытым исходным кодом. (https://github.com/DefectDojo/django-DefectDojo)
Обзор инструмента DefectDojo: почему его выбирают? (https://habr.com/ru/companies/swordfish_security/articles/808615/)

DefectDojo – это Open Source-решение класса ASPM, написанное на Python-фреймворке Django, позволяющее коррелировать результаты сканирований разных инструментов и агрегировать их на одном ресурсе. Этот продукт широко известен среди разработчиков и AppSec-инженеров, поскольку помогает решать множество задач, связанных с управлением уязвимостями.

- проект имеет открытый исходный код;
- проект представляет собой полноценный веб-сервис;
- проект использует базу данных и вспомогательные сервисы;
- проект поддерживает запуск через Docker Compose;
- проект подходит для проверки SAST, DAST и дополнительными security-инструментами.


### Solution Architecture

Этапы безопасного CI/CD-пайплайна для DefectDojo:

1. Получение исходного кода из GitHub.
2. Проверка репозитория на наличие секретов.
3. Статический анализ исходного кода.
4. Проверка зависимостей, конфигураций и контейнерных образов.
5. Сборка и запуск приложения.
6. Динамический анализ запущенного веб-сервиса.
7. Сохранение отчетов сканирования.

##### Components DefectDojo

Основные сервисы DefectDojo описанные в `docker-compose.yml`

| Сервис | Значение |
|---|---|
| nginx | Веб-сервер|
| uwsgi | Основной сервис Django-приложения |
| celerybeat | Планировщик фоновых задач |
| celeryworker | Исполнитель фоновых задач |
| initializer | Первичная инициализация DefectDojo и создание администратора |
| postgres | База данных PostgreSQL |
| valkey | Redis-compatible брокер/кеш для фоновых задач |


P.S.: В `docker-compose.yml` прописаны значения по умолчанию для `DD_SECRET_KEY`, `DD_CREDENTIAL_AES_256_KEY` и пароля PostgreSQL. В продакшене необходимо заменить через переменные окружения или секреты CI/CD.
