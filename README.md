# DevSecOps Pipeline for DefectDojo
## Описание проекта

В рамках проекта был разработан безопасный CI/CD pipeline для open-source веб-приложения DefectDojo.

Цель проекта — внедрение практик DevSecOps в процесс разработки и сопровождения программного обеспечения:
- автоматический анализ исходного кода;
- проверка зависимостей и контейнерных образов;
- поиск секретов в репозитории;
- динамическое тестирование веб-приложения;
- централизованное управление результатами сканирования.


## Используемый проект

В качестве основы выбран:

**DefectDojo**

https://github.com/DefectDojo/django-DefectDojo

DefectDojo — open-source платформа для управления уязвимостями.

Особенности проекта:
- веб-приложение на Django;
- использование PostgreSQL;
- использование Valkey (Redis-compatible cache);
- развертывание через Docker Compose;
- открытый исходный код.


# Общая схема работы

Процесс построения безопасного pipeline включает следующие этапы:

1. Разработчик отправляет изменения в GitHub репозиторий.
2. GitHub Actions запускает CI/CD pipeline.
3. Выполняются проверки безопасности:
   - SAST анализ исходного кода с использованием Semgrep;
   - проверка зависимостей и контейнеров с использованием Trivy;
   - поиск секретов с использованием Gitleaks;
   - DAST анализ веб-приложения с использованием OWASP ZAP.
4. Результаты сканирования импортируются в DefectDojo для централизованного анализа уязвимостей.


## CI/CD Pipeline

Pipeline реализован с использованием GitHub Actions.

При изменении основной ветки выполняются автоматические проверки:

1. Получение исходного кода из GitHub.
2. Запуск проверок безопасности:
   - Gitleaks;
   - Semgrep;
   - Trivy.
3. Формирование отчетов сканирования.
4. Анализ результатов в DefectDojo.
